Bookstore {{ .Chart.Version }} (app {{ .Chart.AppVersion }}) installed as
release "{{ .Release.Name }}" into namespace "{{ include "bookstore.namespace" . }}".

The namespace ENFORCES Pod Security Admission `restricted`. Every workload
this chart renders is restricted-compliant, so all pods are admitted.

What got created (with default values):
  Workloads : catalog, storefront, orders, payments-worker (Deployments),
              redis, rabbitmq (Deployments), postgres (StatefulSet + headless Svc)
  Wiring    : Services, {{ if .Values.ingress.enabled }}Ingress ({{ .Values.ingress.className }}, host {{ .Values.ingress.host }}){{ else if .Values.gateway.enabled }}Gateway API ({{ .Values.gateway.className }}){{ else }}NO external edge (ingress & gateway both off){{ end }}
  Schema    : the db-migrate Job runs as a post-install/post-upgrade HOOK
              AFTER postgres (hook-weight {{ .Values.migrationJob.hookWeight }}); it is idempotent.

Check it:
  helm status {{ .Release.Name }} -n {{ include "bookstore.namespace" . }}
  helm get manifest {{ .Release.Name }} -n {{ include "bookstore.namespace" . }} | less
  kubectl get pods,svc -n {{ include "bookstore.namespace" . }}
  kubectl get job db-migrate -n {{ include "bookstore.namespace" . }}   # the hook
